二分图

匹配/最大独立集/最小覆盖

相关定义

  • 最大匹配：在图中选尽可能多的没有公共点的边，完美匹配即所有点都参与匹配。可以用匈牙利 \(O(nm)\)，也可建模最大流 \(O(m\sqrt n)\)。

  • 最大权匹配：KM 算法 \(O(n^3)\)，费用流 \(O(?)\)。

  • 最大独立集：选尽可能多的没有公共边的点。

  • 最小点覆盖：选尽可能少的点覆盖所有边。

关于匈牙利算法的补充：其实就是不断寻找新的增广路的过程。

研究了几个小时，终于明白了。说穿了，就是你从二分图中找出一条路径来，让路径的起点和终点都是还没有匹配过的点，并且路径经过的连线是一条没被匹配、一条已经匹配过，再下一条又没匹配这样交替地出现。找到这样的路径后，显然路径里没被匹配的连线比已经匹配了的连线多一条，于是修改匹配图，把路径里所有匹配过的连线去掉匹配关系，把没有匹配的连线变成匹配的，这样匹配数就比原来多1个。不断执行上述操作，直到找不到这样的路径为止。——Matrix67《二分图最大匹配问题匈牙利算法》

不难发现最大独立集与最小点覆盖互为对偶问题，最大独立集+最小点覆盖=点数，直观上理解就是两个反着来。

Konig 定理

  • Konig 定理：二分图上，点数-最大匹配=最大独立集
  • 二分图上，最大匹配=最小点覆盖

现在考虑为什么他们能够覆盖所有的边，因为不存在某条边是左边没有标记而右边标记了的（思考一下选出来点是左边标记了的和右边未标记的，所有的边

  1. 要么就是左边有标记（左边选有标记的点）
  2. 要么就是左右都有标记（左边选有标记的点）
  3. 要么就是两端都没有标记（右边选没有标记的点）），因为如果这条边是非匹配边，左端点就可以从这条边到达从而获得标记，而如果它是匹配边，那么就不可能从右边过来，只能是从左边过来的，所以左边肯定打有标记。

应用

一般棋盘类的横行/纵列之类的都很好建模成二分图。

  • P6062 [USACO05JAN]Muddy Fields G

    标准的棋盘类建模。考虑一个点只能被纵/横的木板覆盖。木板肯定是尽可能长的，横木板看成左部，纵木板看成右部。中间的格子就是连边。考虑道我们就是要覆盖所有格子，所以跑一遍最小点覆盖就可以了。

  • UVA1194 Machine Schedule

    可以把任务看作边，两台机子的状态看作点。不难发现也就是一个最小点覆盖，其就等于最大匹配。

  • P3355 骑士共存问题

    不难发现一个骑士与其能到达的点是位于二分图的两部的。对于能互相攻击的位置，连边。跑一遍二分图最大独立集即可。

偏序集-Dilworth 引理

偏序的概念：设 \(A\) 为一个非空集，\(P\) 为 \(A\) 上的一个关系，若 \(P\) 为自反的、反对称的、传递的，则称 \(P\) 为集合 \(A\) 上的一个偏序关系。即：

  • 对于 \(\forall a\in A\) 有 \((a,a)\in P\)
  • 若 \((a,b)\in P\land (b,a)\in P\) 则 \(a = b\)
  • 若 \((a,b)\in P\land (b,c)\in P\) 则 \((a,c)\in P\)

带偏序关系的集合 \(A\) 被称为偏序集。一般使用 \(a\le b\) 表示 \((a,b)\in P\)（这里 \(\le\) 只是一个抽象的符号）。

令 \((X,\le)\) 是一个偏序集，对于 \(a,b\in X\)，若有 \(a\le b\) 或 \(b\le a\)，则称 \(a\) 与 \(b\) 是可比的，否则 \(a\) 和 \(b\) 不可比。一个反链 \(A\) 是 \(X\) 的一个子集，当且仅当其中的任意两个元素都不可比，一个链 \(C\) 是 \(X\) 的一个子集，当且仅当其中的任意两个元素都可比。

Dilworth 引理：最大反链=最小链覆盖，即令 \(m\) 为最长反链的大小，则 \(X\) 可以被划分为 \(m\) 个但不能再少的链。

不难发现对于一个 DAG，我们可以将它抽象成一个偏序集，求出传递闭包就可以找到所有的偏序关系。

此时的反链：对于一个 DAG，定义一个点集 \(S\) 为其反链，当且仅当 \(S\) 中不存在一个点对 \((u,v)\) 使得 \(u\) 可以到达 \(v\)

最小可重链覆盖：用最少的链覆盖完 DAG 中的所有点，边/点均可重复

求最小链覆盖的方法：求传递闭包 + 拆点二分图匹配。

传递闭包即为把所有的偏序关系找出来，使用 Floyd，然后建出一个新的 DAG，跑拆点二分图匹配（网络流 24 题），一开始每个点都是一个链，每匹配一条边，就连起来两个链，需要链的条数就会减一（求传递闭包的目的是把问题转化成每个边只能用一次）

拆点：把每个点 \(u\) 拆成 \(u^+\) 和 \(u^-\)，然后对于原图中的一条边 \((u,v)\)，连边 \((u^+,v^-)\)。拆了点之后跑匹配，这样子就不会有连起来的链了。

最后原来 DAG 点数减去最大匹配就是最小链覆盖的条数。

构造最小链覆盖的方案比较显然，把匹配上的边首尾相连即可。

而构造最长反链的方法即为求出二分图的最大独立集 \(I\)，然后找到所有 \(u^+\) 和 \(u^-\) 都属于 \(I\) 的点 \(u\)，这些点就构成了最长反链 \(A\)。

证明（参考 r_64 的博客）：首先假设原 DAG 有 \(n\) 边，然后求完传递闭包再拆点二分图匹配之后得到的最大匹配为 \(m\)，则不难发现最大独立集的大小为 \(2n - m\)（因为二分图有 \(2n\) 条边）

  • 用反证法不难得到其必然为反链，因为如果存在了 \((u,v)\) 这个偏序关系，那么必然有 \(u^+\) 到 \(v^-\) 的连边，而 \(u^+\) 与 \(v^-\) 在同一个独立集里面，矛盾。

  • 然后考虑证明该反链大小至少为 \(n - m\)，不难发现 $|I| = |A| + $ 满足 \(u^+\in I\) 或者 \(u^-\in I\) 的 \(u\) 的数目，后者不超过 \(n\)，所以前者至少为 \(n - m\)。

  • 最小链覆盖的条数为 \(n - m\)，反链的大小不会超过他，所以我们要求的这个最大反链的大小就是 \(n - m\)。

搞懂了这些之后就可以完成洛谷P4298 [CTSC2008]祭祀。

这道题的题意就是求最大反链大小、构造一个最大反链、每个点有没有可能在构造出来的最大反链里面。对于每个点是否能加进最大反链里面，只需要把这个点、这个点关联的点和边都删掉之后，看答案是否为 \(m - 1\)。

const int maxn = 205;
int n, m;
int f[maxn][maxn], g[maxn][maxn], match[maxn], vis[maxn], idx, to[maxn];
int ban[maxn];
int s[maxn], t[maxn];

int dfs(int u)
{
    if (ban[u]) return 0;
    for (int i = 1; i <= n; ++i)
        if (f[u][i] && vis[i] != idx && !ban[i])
        {
            vis[i] = idx;
            if (!match[i] || dfs(match[i]))
            {
                to[u] = i, match[i] = u;
                return 1;
            }
        }
    return 0;
}

void calc(int u)//进行第二问求解，构造出最大独立集
{
    if (s[u]) return;
    s[u] = 1;
    FOR(i, 1, n)
        if (f[u][i] && !t[i])
            t[i] = 1, calc(match[i]);
    return;
}

int main()
{
    n = read(), m = read();
    FOR(i, 1, m)
        g[read()][read()] = 1;
    FOR(i, 1, n) FOR(j, 1, n) FOR(k, 1, n)
        g[j][k] |= (g[j][i] & g[i][k]);//求传递闭包
    int ans = n;
    memcpy(f, g, sizeof f);
    FOR(i, 1, n) ++idx, ans -= dfs(i);
    printf("%d\n", ans);
    FOR(i, 1, n) if (!to[i]) calc(i);//从未匹配点开始匹配
    FOR(i, 1, n) printf("%d", s[i] && !t[i]);
    puts("");
    FOR(nw, 1, n)
    {
        FOR(i, 1, n) match[i] = to[i] = 0;
        int ret = 0, nn = 0;
        memset(f, 0, sizeof f);
        memset(ban, 0, sizeof ban);
        FOR(i, 1, n)
            if (g[i][nw] || g[nw][i] || i == nw) ban[i] = 1;
            else ++nn;
        ret = nn;
        FOR(i, 1, n)
            FOR(j, 1, n)
                if (!ban[i] && !ban[j]) f[i][j] = g[i][j];
        FOR(i, 1, n) if (!ban[i]) ++idx, ret -= dfs(i);
        printf("%d", ret == ans - 1);
    }
    return 0;
}

网络流

约定：\(n\) 为点数，\(m\) 为边数。

最大流

Dinic 算法

  • 一般图时间复杂度为 \(O(n^2m)\)。
  • 二分图上为 \(O(m\sqrt n)\)。
  • 所有边容量均为 \(1\) 时复杂度为 \(O(\min\{n^\frac 23,\sqrt m\}m)\)。

每次增广前使用 bfs 给图分层。每个点的层数为其到源点的最近距离。作用：

  1. 判断图中是否还有增广路；
  2. 使得增广路最短。

然后 dfs 找增广路。每次增广都只往层数比当前点多 \(1\) 的点增广，保证找到的增广路最短。

两个必须加的优化：

  1. 当前弧优化：如果一个边被增广过，就没有必要被增广第二次。下一次就没必要经过这些已经被增广的边。
  2. 多路增广：用没用完的残余流量继续找出一条增广路。这样一次 dfs 就找到多条增广路。
  3. 炸点优化：一个点没有办法产生贡献了就把它炸掉。

using namespace fastIO;

template<typename T> il T min(const T &a, const T &b) {return a < b ? a : b;}
template<typename T> il T max(const T &a, const T &b) {return a > b ? a : b;}

const int maxn = 205, maxm = 5005;

typedef long long ll;
const ll INF = 1e18;

struct edge {
    int to, nxt;
    ll w;
} e[maxm << 1];

int head[maxn], dep[maxn], cur[maxn], cnt = 1;
int n, m, s, t;

il void add(int u, int v, ll w) {
    e[++cnt].to = v;
    e[cnt].w = w;
    e[cnt].nxt = head[u];
    head[u] = cnt;
    return;
}

bool bfs() {
    static int q[maxn], qhead, qtail;
    memset(dep, -1, sizeof dep);
    memcpy(cur, head, sizeof head);
    q[qhead = qtail = 1] = s;
    dep[s] = 0;
    while (qhead <= qtail) {
        int u = q[qhead++];
        GO(u, head) {
            if (e[i].w > 0 && dep[v] == -1) {//增广
                q[++qtail] = v;
                dep[v] = dep[u] + 1;
                if (v == t) return true;//如果找到增广路了退出
            }
        }
    }
    return false;
}

ll dfs(int u, ll in) {
    if (u == t) return in;
    ll out = 0;
    GO(u, cur) {//当前弧优化
        cur[u] = i;
        if (e[i].w > 0 && dep[v] == dep[u] + 1) {
            ll res = dfs(v, min(in, e[i].w));
            e[i].w -= res, e[i ^ 1].w += res;
            in -= res, out += res;
        }
        if (!in) break;
    }
    if (!out) dep[u] = -1;//炸点优化
    return out;
}

ll dinic() {
    ll ret = 0;
    while (bfs()) ret += dfs(s, INF);
    return ret;
}

int main() {
    read(n), read(m), read(s), read(t);
    FOR(i, 1, m) {
        int u, v; ll w; read(u), read(v), read(w);
        add(u, v, w), add(v, u, 0);
    }
    print(dinic());
    return output(), 0;
}

最小割

最大流=最小割。

  • 简单割：割集的每条边都与 \(S\) 或 \(T\) 相连。

集合划分问题

P2057 [SHOI2007]善意的投票 / [JLOI2010]冠军调查

有两个集合 \(S\) 和 \(T\) 和 \(n\) 个物品。

将 \(i\) 物品放入集合 \(S\) 花费 \(a_i\)，放入 \(T\) 花费 \(b_i\)。

有若干个限制条件 \((u, v, w)\) 表示 \(u\) 和 \(v\) 若不在一个集合则代价为 \(w\)。

每个物品最多且必须在两集合之一，求最小代价。

设源点 \(S\) 和汇点 \(T\)。\(i\) 号点从 \(S\) 向 \(i\) 连 \(a_i\) 容量的边，从 \(i\) 向 \(T\) 连 \(b_i\) 容量的边。对于 \((u, v, w)\) 在 \(u\) 和 \(v\) 间连容量为 \(w\) 双向边。

当 \(S\) 和 \(T\) 被割开时，\(S\) 可达 \(i\) 代表 \(i\) 放入 \(S\)，\(i\) 可达 \(T\) 代表 \(i\) 放入 \(T\)。

割开 \((S, i)\) 即为 \(i\) 放入 \(T\)，割开 \((i, T)\) 即为 \(i\) 放入 \(S\)。割开 \((u, v)\) 代表 \(u\) 和 \(v\) 不在一个集合。

跑最小割就可以了。

相关习题：

  • SP839 OPTM - Optimal Marks 拆位考虑。
  • P1361 小M的作物 打包型问题，建虚点。

最小割的可行边和必须边

P4126 [AHOI2009]最小割

首先，如果一条边没有满流，那么它肯定不可能在最小割里面，因为总有更好的满流边能替代他。

然后，在残余网络中如果找得到 \((u, v)\)，那么 \((u, v)\) 就不可能是瓶颈，所以不可能出现在最小割里面（相当于割了 \((u, v)\) 之后没用，还是联通的）。

而如果 \(S\) 在残余网络中能到达 \(u\)，\(v\) 在残余网络中能到达 \(T\)，则 \((u, v)\) 一定是必选的，因为 \((u, v)\) 已经是瓶颈了。

具体是实现就是求一下残余网络的 SCC 就行了。

平面图最小割

边与边只在顶点相交的图叫做平面图。

注意到平面图的边划分出了若干个区域，将每个区域抽象成一个点，原图中边两侧的区域对应的点连边，则得到平面图对应的对偶图。

平面图最小割等于对偶图最短路。感性理解一下就好了。

P4001 [ICPC-Beijing 2006]狼抓兔子。

P2046 [NOI2010] 海拔。比较毒瘤，题解。

最小割的最小边数

首先有一个非常显然的结论：边权为 \(1\) 的图的最小割边数等于最小割。

那么我们考虑将边权不为 \(1\) 的图进行转化：这里是一个套路，将所有边权乘上一个不影响答案的大数 \(E\)（至少大于边数）再加 \(1\)，跑出最大流 \(F\)，则 \(F/E\) 为最小割，\(F\bmod E\) 为最小割边数。证明略了，挺显然的。

还有一种做法是跑一遍最大流，然后将残量网络中的边权设为 \(\inf\)，满流边的边权设为 \(1\)，再跑一遍最大流。这样子做的正确性是相当于重新建了张图，然后求新图的最小割（即为边数）。

最大权闭合子图

  • 闭合图：有向图 \(G= (V, E)\) 的一个点集，且该点集的出边指向的点仍在该点集中。
  • 最大权闭合图：给每个点赋一个权值，点权最大的闭合图。

反映了事件间必要条件的关系。一个事件的发生需要其所有前提发生才发生。最大权闭合图对应了收益最大的事件选择。

构图：建源汇点 \(S\) 和 \(T\)，\(S\) 连向所有正权点，容量为点权；其余点连向 \(T\)，容量为点权的绝对值，对于原图中的边 \((x, y)\)，连边 \((x, y, +\infty)\)。

  • 该网络 \(N\) 的最小割为简单割。

    容量为 \(+\infty\) 的边不可能被割断。

  • （可行性）该网络中的简单割与闭合图一一对应。设简单割将 \(V\) 化成了\(V_1\) 和 \(V_2\) 两个点集，则含有 \(S\) 的点集 \(V_1\) 为闭合图。

    理解：原图的边容量为 \(\infty\)，所以能走的话一定会走到底的。不存在 \(V_1\) 中有点的出边连向 \(V_2\)。

  • （最优性）最小割对应的 \(V_1\) 为最大权闭合图。

    设 \(V_1\) 中正权点点集为 \(V_1^+\)，同理定义 \(V_1^-\)，\(V_2^+\) 和 \(V_2^-\)。设最小割容量为 \(C\)。

    那么不难发现 \[ C = \sum_{i\in V_2^+}w_i - \sum_{i\in V_1^-}w_i \] 然后，设 \(V_1\) 对应的闭合图的权值为 \(W\)，则 \[ W = \sum_{i\in V_1^+}w_i + \sum_{i\in V_1^-}w_i \] 然后 \[ \begin{aligned} C + W &= \sum_{i\in V_1^+}w_i + \sum_{i\in V_1^-}w_i + \sum_{i\in V_2^+} - \sum_{i\in V_1^-}w_i\\ C + W&= \sum_{i\in V_1^+} w_i + \sum_{i\in V_2^+}w_i\\ W &= \sum_{i\in V^+}w_i - C \end{aligned} \] 考虑到 \(\sum_{i\in V^+}w_i\) 为定值，故 \(C\) 最小时 \(W\) 最大，证毕。

所以，建出图后求出最小割 \(C\)，然后用所有正权点的点权和减去最小割即得到最大权闭合图的点权和。

输出方案：最小割对应满流的边，所以从 \(S\) 出发找残量网络即能找到对应的点。

template<typename T> il T min(const T &a, const T &b) {return a < b ? a : b;}
template<typename T> il T max(const T &a, const T &b) {return a > b ? a : b;}

typedef long long ll;
const ll INF = 1e18;
const int maxn = 5e3 + 5, maxm = 5e4 + 5;

struct edge {
    int to, nxt;
    ll w, c;
} e[maxm << 1];

int head[maxn], cnt = 1, s, t;

il void add(int u, int v, ll w, ll c) {
    e[++cnt].to = v, e[cnt].w = w, e[cnt].c = c;
    e[cnt].nxt = head[u], head[u] = cnt;
    e[++cnt].to = u, e[cnt].w = 0, e[cnt].c = -c;
    e[cnt].nxt = head[v], head[v] = cnt;
    return;
}

namespace MCMF {
ll dis[maxn], flow[maxn], maxflow, mincost;
int pre[maxn], inq[maxn];

bool SPFA() {
    memset(inq, 0, sizeof inq);
    memset(dis, 0x3f, sizeof dis);
    std::queue<int> q;
    q.push(s);
    inq[s] = 1, dis[s] = 0;
    flow[s] = INF;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        inq[u] = 0;
        GO(u, head) {
            if (e[i].w > 0 && dis[u] + e[i].c < dis[v]) {
                dis[v] = dis[u] + e[i].c;
                flow[v] = min(flow[u], e[i].w);
                pre[v] = i;
                if (!inq[v]) {
                    inq[v] = 1;
                    q.push(v);
                }
            }
        }
    }
    return dis[t] < INF;
}

void update() {
    int now = t;
    while (now != s) {
        int i = pre[now];
        e[i].w -= flow[t], e[i ^ 1].w += flow[t];
        now = e[i ^ 1].to;
    }
    maxflow += flow[t];
    mincost += flow[t] * dis[t];
    return;
}

void MCMF() {
    while (SPFA()) update();
    return;
}
} // namespace MCMF

int n, m;

int main() {
    read(n), read(m), read(s), read(t);
    FOR(i, 1, m) {
        int u, v, w, c;
        read(u), read(v), read(w), read(c);
        add(u, v, w, c);
    }
    MCMF::MCMF();
    print(MCMF::maxflow), putchar(' '), print(MCMF::mincost);
    return output(), 0;
}

二分图博弈

感谢这篇文章。

基本定义

博弈人数两人，轮流决策。

状态可被分为二分图的左部和右部两个集合，转移只发生在两个集合之间。

不可以访问重复状态。

无法转移者被判负。

结论及证明

设决策起点为 \(P\)，若二分图的最大匹配一定包含 \(P\)，则先手必胜，否则必败。

分如下情况讨论：

  1. 如果最大匹配不一定包含 \(P\)：

    先手首先走到匹配点上（否则会产生新的匹配，与最大匹配矛盾），接下来后手走的边必为匹配点（否则这条路是一条增广路，和最大匹配矛盾）。最后肯定是先手无路可走。先手必败。

  2. 如果最大匹配一定包含 \(P\)：

    先手一直走匹配边，后手必败。

  3. 如果 \(P\) 为孤立点：

    显然先手必败。

判断方法

判断 \(P\) 是否为先手必胜的决策起点，最简单的方法肯定是先把 \(P\) 抠掉然后跑最大流，然后再把 \(P\) 点加回去在残量网络上继续跑，若最大流增加，则 \(P\) 一定在最大匹配里面。

然而有些题目需要求出所有的合法起点，枚举的时间复杂度铁定是超时的。考虑一个点何时不一定在最大匹配里面：考虑匈牙利算法，无非就是产生了一条交替路。记源汇分别为 \(S\) 和 \(T\)，分别相连的点集为 \(X\) 和 \(Y\)，那么不一定在最大匹配里面的点就是：

  • 从 \(S\) 出发每次找非满流边 dfs，能走到的所有 \(X\) 里面的点。
  • 从 \(T\) 出发每次找满流边 dfs，能走到的所有 \(Y\) 里面的点。

实际上这两种都是找交替路的过程。第一种等价于找非匹配-匹配-非匹配……的边构成的路径，第二种等价于找匹配-非匹配-匹配……的边构成的路径。

例题

P4055 [JSOI2009]游戏

传送门。大意：给定一张网格，有些点不能走，先手首先选择起点，然后后手和先手轮流移动到可以走的四联通格子，格子不能重复经过。求所有使得先手必胜的起点。

发现对于 \((i, j)\)，\(i + j\) 的奇偶性可以天然的将整张图的所有格点划分成一张二分图。然后问题显然变为二分图博弈的问题。如果一定在最大匹配里面的点一定是赢不了的点。

所以按照刚才说的方式 dfs 就可以了。
