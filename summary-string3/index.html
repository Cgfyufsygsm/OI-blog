前言

后缀数组和后缀自动机，写的会比较毛躁。后缀平衡树和后缀树不会，不写。

SA 基础

SAM 基础

很牛逼的后缀结构，建立在 \(S\) 之上的 SAM 是一个只接受 \(S\) 的后缀的 DFA。

和 SA 没啥关系，并且构建复杂度为线性。

定义

字符串 \(S\) 的 SAM 是一个接受 \(S\) 的所有后缀的最小 DFA。

直观地理解的话：

  • SAM 是一张 DAG，节点叫做状态，边即为转移。
  • 存在初始状态 \(t_0\)。
  • 一个转移对应一个字母。
  • 存在若干终止状态，从 \(t_0\) 到一个终止状态的路径对应了 \(S\) 的一个后缀。
  • 在所有满足上述状态的 DFA 中大小最小。

同时，由于一条从 \(t_0\) 到终止状态的路径总能对应一个后缀，那么不难发现任何一条从 \(t_0\) 出发的路径都能对应 \(S\) 的一个子串（可理解为后缀的前缀），注意到一个状态可能对应多条路径。

endpos（right 集合）

所有子串是 \(O(n^2)\) 级别的，所以 SAM 是高度压缩了的结构。

那么我们有必要将一些子串划分为一个等价类存储。

定义：一个子串的 \(\operatorname{endpos}\) 为其在原串中结束位置的集合。

e.g. \(s = \texttt{abcabcb}\)，则 \(\operatorname{endpos}(\texttt{ab}) = \{2, 5\}\)，\(\operatorname{endpos}(c) = \{3, 6\}\)。

定义：\(\operatorname{endpos}\) 相同的若干子串构成一个等价类。

SAM 的每个状态对应一个 \(\operatorname{endpos}\) 等价类。

下面考虑两个子串 \(S_1\) 和 \(S_2\)，且假设 \(|S_1|\le |S_2|\)。

引理 \(1\)：\(S_1\) 是 \(S_2\) 的后缀 \(\iff\) \(\operatorname{endpos}(S_2)\subseteq \operatorname{endpos}(S_1)\)。

证明：能找到 \(S_2\) 结尾的地方必定会出现 \(S_1\)。

引理 \(2\)：要么 \(\operatorname{endpos}(S_1)\cap \operatorname{endpos}(S_2) = \varnothing\)，要么 \(\operatorname{endpos}(S_2)\subseteq \operatorname{endpos}(S_1)\)，取决于 \(S_2\) 是否为 \(S_1\) 的后缀。

证明：只要 \(\operatorname{endpos}(S_2)\cap \operatorname{endpos}(S_1)\not=\varnothing\)，\(S_1\) 就一定为 \(S_2\) 的后缀，即根据引理 \(1\)。

引理 \(3\)：对于一个 \(\operatorname{endpos}\) 等价类，其中的子串按长度非递增的顺序排序，则后一个必为前一个的前缀。该等价类中子串的长度恰好覆盖 \([x, y]\)。

证明：对于只有一个元素的等价类，成立。

由引理 \(1\)，等价类中无长度相等的字符串。设 \(S_1\) 为等价类中最短的串，\(S_2\) 为等价类中最长的串。考虑长度在 \((|S_1|, |S_2|)\) 中的真后缀，发现这些后缀也在同一个等价类中，显然成立。

后缀链接（link）和 parent tree

如果我们在一个等价类的最长子串 \(S_1\) 的前面添加一个字母形成 \(S_2\)，那么其必然不属于这个等价类，但是 \(S_2\) 为 \(S_1\) 的子串，故有 \(\operatorname{endpos}(S_2)\subset \operatorname{endpos}(S_1)\)，即添加的字符越多，\(\operatorname{endpos}\) 等价类越小。用这个子集关系可以建立一棵树，称为 parent tree。如图是 \(\texttt{aababa}\) 的 parent tree。红色即为该 \(\operatorname{endpos}\) 等价类包含的子串，黑色为 \(\operatorname{endpos}\) 的结束位置。

这个树的节点和 SAM 的节点意义相同。这棵树首先必然包含所有的子串，又由引理 \(2\)，每个子串不可能被分到两个等价类里面。

引理 \(4\)：parent tree 的节点数为 \(O(n)\)。

证明：加入一个字符最多将一个等价类分为两半，经过 \(n - 1\) 个分割能把原来的 \(n\) 个都分散，所以最多为 \(2n -1\) 个节点。

考虑一个状态 \(u\)，其代表了一个 endpos 等价类，定义 \(\operatorname{longest}(u)\) 表示其最长的子串，那么不难发现其他所有子串都是它的后缀。并且其还有一部分后缀在其他等价类内。记 \(t\) 为这样的最长后缀，将 \(u\) 的后缀链接连接到 \(t\) 上。

即，后缀链接 \(\operatorname{link}(u)\) 连接到对应于 \(\operatorname{shortest}(u)\) 的最长后缀的状态 \(t\)。可以发现其形成了树形结构。

引理 \(5\)：后缀链接形成的树与使用 endpos 子集关系构造的树相同。

可以得到 \(\mathrm{endpos}(u)\subsetneq\mathrm{endpos}(\mathrm{link}(u))\)（因为若相等那么其应该是一个节点才对）。

所以，总结一下一些对于解决问题很有用的性质：

  • \(S\) 的子串由其结束的位置被划分为若干个 \(\operatorname{endpos}\) 等价类。
  • SAM 由初始状态 \(t_0\) 和每个等价类对应的状态组成。
  • 对于一个状态 \(u\)，其对应一个或多个子串。记 \(\operatorname{len}(u)\) 为其最长子串 \(\operatorname{longest}(u)\) 的长度，\(\operatorname{minlen}(u)\) 为其最短子串的长度，则对应这个状态的子串都为其最长子串的某个后缀，且长度覆盖了区间 \([\operatorname{minlen}(u), \operatorname{len}(u)]\) 的每一个整数。
  • 对于除了 \(t_0\) 外的状态 \(u\)，定义后缀链接为连接到某状态的最长子串的长度为 \(\operatorname{minlen}(u) - 1\) 的后缀的一条边。从 \(t_0\) 出发的后缀链接可形成一棵 parent tree，对应了 \(\operatorname{endpos}\) 集合的包含关系。
  • 定义 \(\operatorname{link}(u)\) 为 \(u\) 在 parent tree 的父亲（即后缀链接），则 \(\operatorname{minlen}(u) = \operatorname{len}(\operatorname{link}(u)) + 1\)。

在线构造

\(O(n)\) 在线构造 SAM，每次加一个字符，正确性我懒得纠结了，有点神秘，背住就行。

struct node {
    int ch[26], len, fa;
    node() {memset(ch, 0, sizeof ch), len = fa = 0;}
} sam[maxn << 1];

int las = 1, tot = 1;//（t_0 为 1 号点）

void add(int c) {
    int cur = ++tot;
    sam[cur].len = sam[las].len + 1;
    int p = las;
    while (p && !sam[p].ch[c]) sam[p].ch[c] = cur, p = sam[p].fa;
    if (!p) sam[cur].fa = 1;
    else {
        int q = sam[p].ch[c];
        if (sam[q].len == sam[p].len + 1)
            sam[cur].fa = q;
        else {
            int clone = ++tot;
            sam[clone] = sam[q];
            sam[clone].len = sam[p].len + 1;
            while (p && sam[p].ch[c] == q) sam[p].ch[c] = clone, p = sam[p].fa;
            sam[q].fa = sam[cur].fa = clone;
        }
    }
    las = cur;
    return;
}

大概解释一下：判断 \(\operatorname{len}(q) = \operatorname{len}(p) + 1\) 的目的是：\(\operatorname{longest}(p) + c\) 为新串的后缀，若该条件成立，则 \(\operatorname{longest}(q) = \operatorname{longest} + c\)，\(q\) 也为新串的后缀，且他们的 \(\operatorname{endpos}\) 都增加了 \(n\)。所以把 cur 的 fa 设为 q。若 \(\operatorname{longest}(q)\) 不为新后缀。所以我们需要将 q 进行分裂，一部分包含比新后缀更长的串，一部分包含新后缀以及比新后缀短的串。

广义 SAM

正版广义 SAM 是建立在 Trie 树上的，我也懒得研究了，但是特判 \(las\) 是否有指向其的节点是很好背的。

struct Node {
    int ch[26], fa, len;
} sam[maxn];

int tot = 1, las = 1;

void extend(int c) {
    auto split = [&c](int p, int q) {
        int clone = ++tot;
        sam[clone] = sam[q];
        sam[clone].len = sam[p].len + 1;
        while (p && sam[p].ch[c] == q) sam[p].ch[c] = clone, p = sam[p].fa;
        return sam[q].fa = clone;
    };
    if (sam[las].ch[c]) {
        int q = sam[las].ch[c];
        if (sam[q].len == sam[las].len + 1) las = q;
        else las = split(las, q);
        return;
    }
    int cur = ++tot;
    sam[cur].len = sam[las].len + 1;
    int p = las;
    while (p && !sam[p].ch[c]) sam[p].ch[c] = cur, p = sam[p].fa;
    if (!p) sam[cur].fa = 1;
    else {
        int q = sam[p].ch[c];
        if (sam[q].len == sam[p].len + 1) sam[cur].fa = q;
        else sam[cur].fa = split(p, q);
    }
    las = cur;
    return;
}

就是类似的分裂流程，仔细想想会发现很有道理。

其具有普通 SAM 该有的所有性质。

SAM 应用

在处理子串和后缀问题的时候，SAM 是非常强力的工具。

处理子串信息

  • 子串出现次数：endpos 集合的大小而已。
  • 子串长度：记得你覆盖了 \([\operatorname{minlen}(u), \operatorname{len}(u)]\) 里面的每一个长度，而 \(\operatorname{minlen}(u) = \operatorname{len}(\operatorname{link}(u)) + 1\)。
  • 子串长度出现次数：结合上面两个随便做。

对于 endpos 集合大小的计算，将每个前缀对应的点的 \(size\) 设为 \(1\)（注意到他们肯定是 parent tree 的叶子节点），然后直接树形 dp 即可。当然也可以按照 \(\mathrm{len}(u)\) 基排一遍后非递归做，因为父亲的 \(\mathrm{len}\) 一定小于儿子的。

例题 \(1\)：P3804 【模板】后缀自动机。

求出 \(S\) 的所有出现次数不为 \(1\) 的子串的出现次数乘上该子串长度的最大值。

#include <cstdio>
#include <cstring>
#define FOR(i, a, b) for (int i = a; i <= b; ++i)
#define il inline

const int maxn = 1e6 + 5;

char s[maxn];
int len;

struct node {
    int ch[26], len, fa, size;
    node() {memset(ch, 0, sizeof ch), len = fa = size = 0;}
} sam[maxn << 1];

il int max(int a, int b) {return a > b ? a : b;}

int las = 1, tot = 1, ans;
int head[maxn << 1], to[maxn << 2], nxt[maxn << 2], cnte;

void add(int u, int v) {
    to[++cnte] = v;
    nxt[cnte] = head[u];
    head[u] = cnte;
    return;
}

void add(int c) {
    int cur = ++tot;
    sam[cur].len = sam[las].len + 1;
    int p = las;
    while (p && !sam[p].ch[c]) sam[p].ch[c] = cur, p = sam[p].fa;
    if (!p) sam[cur].fa = 1;
    else {
        int q = sam[p].ch[c];
        if (sam[q].len == sam[p].len + 1)
            sam[cur].fa = q;
        else {
            int clone = ++tot;
            sam[clone] = sam[q];
            sam[clone].len = sam[p].len + 1;
            while (p && sam[p].ch[c] == q) sam[p].ch[c] = clone, p = sam[p].fa;
            sam[q].fa = sam[cur].fa = clone;
        }
    }
    las = cur;
    return;
}

void dfs(int u) {
    for (int i = head[u]; i; i = nxt[i]) {
        int v = to[i];
        dfs(v);
        sam[u].size += sam[v].size;
    }
    if (sam[u].size != 1)
        ans = max(ans, sam[u].len * sam[u].size);
    return;
}

int main() {
    scanf("%s", s + 1), len = strlen(s + 1);
    FOR(i, 1, len) add(s[i] - 'a');
    int p = 1;
    FOR(i, 1, len) p = sam[p].ch[s[i] - 'a'], sam[p].size = 1;
    FOR(i, 2, tot) add(sam[i].fa, i);
    dfs(1);
    printf("%d\n", ans);
    return 0;
}

不同子串个数

建立 SAM，每个子串相当于 \(t_0\) 开始的一条路径，所以设 \(f_u\) 为从 \(u\) 开始的路径条数，则 \(f_u = 1 + \sum_vf_v\)。（\(u\) 到自己的也算），于是乎答案为 \(f_1 - 1\)（空串最后不考虑）。

还有另外一种理解方式：SAM 每个节点表示的串没有交，而 \[ \operatorname{minlen}(u) = \operatorname{len}(\operatorname{link}(u)) + 1 \] 所以统计 \(\sum_{u}(\operatorname{len}(u) - \operatorname{len}(\operatorname{link}(u)))\) 即可。

void add(int c) {

例题 \(9\)：#6071. 「2017 山东一轮集训 Day5」字符串

给定 $ n $ 个字符集为小写字母的字符串 $ s_i $，一个串 $ t $ 是可接受的，当且仅当 $ t $ 可以表示成 $ p_1 + p_2 + + p_n $，其中 $ p_i $ 为 $ s_i $ 的一个子串（可以为空），$ + $ 表示字符串的拼接。问有多少种本质不同的字符串 $ t $ 是可接受的。答案对 $ 10 ^ 9 + 7 $ 取模，$ n ^ 6,_{i = 1} ^ n |s_i| ^ 6 $。

发现这样可以不重不漏计数（不会出现一个串可以通过两个路径走的方式，因为优先在本串内转移）。

例题 \(13\)：CF235C Cyclical Quest

给定一个主串 \(S\) 和 \(n\) 个询问串，求每个询问串的所有循环同构在主串中出现的次数总和，\(|S|,\sum |x_i|\le 10^6\)，\(n\le 10^5\)。

直接拿每个询问串在主串上跑匹配，断环为链，然后到了长度要强制失配，记得本质不同打标记。

FOR(kase, 1, q) {
    cin >> (s + 1);
    int n = strlen(s + 1);
    int len = 0, u = 1;
    auto extend = [&](int c) {
        for (; u && !sam[u].ch[c]; u = sam[u].fa) len = sam[sam[u].fa].len;
        if (sam[u].ch[c]) ++len, u = sam[u].ch[c];

        if (len == n && sam[u].vis < kase) ans += sam[u].size, sam[u].vis = kase;
    }
    cout << ans << endl;
}

例题 \(14\)：P3763 [TJOI2017]DNA

\(|\Sigma| = 4\)，给定 \(S\) 和 \(S_0\)（长度不超过 \(10^5\)），问 \(S_0\) 中有多少个与 \(S\) 等长的子串满足其修改不超过 \(3\) 个位置后等于 \(S\)。

FFT 做法在此不表，考虑建出 \(S_0\) 的 SAM 然后直接在上面开始匹配，但是需记录状态（失配次数）。这样最多遍历每个 SAM 节点的 \(4\) 种状态，复杂度 \(O(n)\)。

例题：P6640 [BJOI2020] 封印

给出只包含小写字母 \(a,b\) 的两个字符串 \(s, t\)，\(q\) 次询问，每次询问 \(s[l..r]\) 和 \(t\) 的最长公共子串长度。\(|s|,|t|,q\le 2\times 10^5\)。

建 \(t\) 的 SAM，拿 \(s\) 在上面跑不难得到对于每个前缀 \(s[1..i]\)，其能匹配上的最长后缀 \(f_i\)。那么答案显然为 \(\max\limits_{i=l}^r\{\min(f_i, i - l + 1)\}\)。

对于这个式子可以二分答案 \(mid\)，然后必然有 \(i-l+1\ge mid\)，所以直接在 \([l+mid-1, r]\) 查最大值就行了，\(i-l+1\) 的约束直接被扬了。

第 \(k\) 小子串

例题 \(15\)：P3975 [TJOI2015]弦论

求本质不同/位置不同的 \(k\) 小子串

DP 处理出每个节点的子串数量，然后类似线段树上二分一样的直接在 SAM 上 dfs 就可以了。

int buc[maxn], a[maxn << 1];

void print(int x, int k) {
    if (k <= size[x]) return;
    k -= size[x];
    FOR(c, 0, 25) {
        int y = sam[x].ch[c];
        if (!y) continue;
        if (k > sum[y]) {
            k -= sum[y];
            continue;
        }
        putchar(c + 'a'); print(y, k); return;
    }
}

int main() {
    scanf("%s", s + 1);
    scanf("%d %d", &t, &k);
    len = strlen(s + 1);
    FOR(i, 1, len) add(s[i] - 'a');

    FOR(i, 1, tot) ++buc[sam[i].len];
    FOR(i, 1, len) buc[i] += buc[i - 1];
    FOR(i, 1, tot) a[buc[sam[i].len]--] = i;
    DEC(i, tot, 1) size[sam[a[i]].fa] += size[a[i]];
    FOR(i, 1, tot) t ? (sum[i] = size[i]) : (sum[i] = size[i] = 1);
    size[1] = sum[1] = 0;
    DEC(i, tot, 1) // 考虑 SAM 的 1..tot 是天然的拓扑序
        FOR(c, 0, 25)
            if (sam[a[i]].ch[c])
                sum[a[i]] += sum[sam[a[i]].ch[c]];
    if (sum[1] < k) puts("-1");
    else print(1, k);
    return 0;
}

最长公共子串

注意子串是连续的。

例题 \(16\)：SP1811 LCS - Longest Common Substring

求 \(S\) 和 \(T\) 的最长公共子串。

建出 \(S\) 的 SAM，然后拿 \(T\) 在 \(S\) 上跑匹配，能匹配则 ++tmp，否则在 parent 树往上跳，匹配长度变为有出边的节点的 len + 1。

int ans = 0, len = 0, p = 1;
for (auto ch : t) {
    int c = ch - 'a';
    if (!sam[p].ch[c]) {
        while (p != 1 && !sam[p].ch[c]) p = sam[p].fa;
        len = sam[p].len;
    } 
    if (sam[p].ch[c]) {
        p = sam[p].ch[c];
        ++len;
    }
    ans = max(ans, len);
}
cout << ans << endl;

例题 \(17\)：SP1812 LCS2 - Longest Common Substring II

求若干个字符串的 LCS，字符串只有 \(10\) 个。

建出广义 SAM，然后对每个节点维护一下其能被多少串经过，在 parent tree 上做一下向上的并，若对应状态包含所有串则用 \(\mathrm{len}(u)\) 更新答案。

例题：CF1037H Security

给定文本串 \(S\)，\(q\) 次询问。每次询问给定 \(l,r\) 和子串 \(T\)，问 \(S[l..r]\) 中字典序严格大于 \(T\) 的字典序最小的子串，没有输出 \(-1\)。

不考虑 \([l,r]\) 的限制的话，可以直接贪心。假设在 \(T[p]\) 处失配，就可以趁机让字典序大起来了，直接挑最小有的且 \(>T[p]\) 的字符接上去就行，不行就跳 fail，跳到根还不行就寄寄。

那么如果需要考虑 \([l,r]\) 的话，发现就是 endpos 的问题，endpos 要在 \([l + len - 1, r]\) 里面有，其中 \(len\) 是当前串长。

代码其实应该不太难写。

例题：P4094 [HEOI2016/TJOI2016]字符串

数据范围 \(10^5\)，给定 \(S\)，\(m\) 次询问 \(S[a..b]\) 的子串与 \(S[c..d]\) 的 LCP 的最大值。

反转一下，把求 LCP 变成求最长公共后缀。然后二分一个 \(len\)，变成求 \(S[d - len + 1..d]\) 在 \(S[a..b]\) 中是否存在。

似曾相识的画风，处理在 \([a,b]\) 中是否出现明显可以维护 edp。但是对于前面的 \([d - len + 1, d]\) 的限制的话，我们可以考虑一开始预处理 \(S[1..i]\) 对应的状态，然后 fail 树往回跳来使得左端点到达 \(d - len + 1\)，即要求 \(\mathrm{len}(u)\) 恰好大于 \(len\)，跳这个过程就倍增，时间复杂度俩 \(\log\)。
