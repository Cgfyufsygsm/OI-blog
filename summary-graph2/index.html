同余最短路

Kruskal 重构树

定义

考虑 Kruskal 算法的过程，我们会从小到大加边。考虑依此构建出一棵“重构树”。具体地：

  • 新建 \(n\) 个集合，每个集合初始有一个点，权为 \(0\)。
  • 对于 Kruskal 的每一次成功的加边，都建一个辅助点，点权为对应边的边权，同时将边端点所在的两个集合的根节点连向辅助点，然后将其合并成一个集合，辅助点为根。
  • 这样 \(n - 1\) 轮下来我们会得到恰有 \(n\) 个叶子节点的二叉树，同时每个非叶子节点都恰好有两个儿子。这棵树就叫做 Kruskal 重构树。

性质

以下性质基于依照“最小生成树”构建的重构树：

  • 该树为一个大根堆。
  • 对于原图中任意两点 \(x\)、\(y\)，其所有简单路径上最大边权的最小值 = 最小生成树上两点间简单路径的最大值 = 重构树上两者 LCA 的权值。
  • 同理，到 \(x\) 的简单路径上最大边权的最小值 \(\le v\) 的所有点 \(y\) 均在 Kruskal 重构树上的某棵子树内，且恰好为这棵子树的所有叶子节点。倍增找到点权 \(\le v\) 的 \(x\) 的最浅的祖先即可。

Kruskal 重构树一般用于解决瓶颈路一类的问题。

应用

例题 \(1\)：P7834 [ONTAK2010] Peaks 加强版

给定一张 \(n\) 个点、\(m\) 条边的无向图，第 \(i\) 个点的权值为 \(a_i\)，边有边权。

有 \(q\) 组询问，每组询问给定三个整数 \(u, x, k\)，求从 \(u\) 开始只经过权值 \(\leq x\) 的边所能到达的权值第 \(k\) 大的点的权值，如果不存在输出 \(-1\)，\(1 \leq n \leq 10^5\)，\(0 \leq m, q \leq 5 \times 10^5\)，\(1 \leq a, b \leq n\)，\(1 \leq a_i, v \leq 10^9\)，\(0 \leq u', x', k' < 2^{31}\)。强制在线。

离线版本貌似可以用线段树合并之类的技巧直接水过，注意到询问的是类似于瓶颈路的形式，考虑 Kruskal 重构树。首先候选点一定在某个子树内，那么先倍增找到这个子树的根节点，然后求子树内第 \(k\) 大的叶子节点可以拿主席树啥的随便水水就过了。

设定较长，见原题面。

首先注意到我们要最小化的是步行的路程，所以预先处理所有节点到 \(1\) 的 \(l\) 意义下的最短路。

那么剩下的问题就变为，求出从 \(v\) 开始只经过权值 \(a \ge S\) 的边，能到达的点的 \(dis\) 的最小值。那么就按最大生成树构造 Kruskal 重构树然后预处理子树内 \(dis\) 最小值即可。

最小直径生成树

最小直径生成树，指的是一张无向图的所有生成树中，直径最小的那一棵生成树。

图的绝对中心可以存在于一条边上或某个结点上，该中心到所有点的最短距离的最大值最小。根据其定义，到绝对中心距离最远的节点至少有 \(2\) 个。

考虑用全源最短路算法（Floyd 或 \(n\) 轮 Dijkstra）求出两两点对间的最短路 \(d(i, j)\)，\(rk(i, j)\) 记录 \(i\) 到其他节点中第 \(j\) 小的点。

  • 图的绝对中心可能在某个点上，考虑用离距离预选节点最远的节点来更新答案，即 \[ ans\leftarrow\min(ans, d(i, rk(i, n))\times 2) \] 该情况比较简单。

  • 图的绝对中心也可能在某条边上，若在边 \((u, v, w)\) 上取距离 \(u\) 为 \(x\) 的点作为预选点，则点 \(i\) 到预选点的距离则为 \(\min(d(u, i) + x, d(v, i) + w - x)\)。这是一个分段函数（并且先上升后降低，斜率绝对值相同），然后对于图上的所有节点，如下图（来自 OI-Wiki）：

    这些折线交点中的最低点的横坐标就是图的绝对中心的位置。

    考虑如何求这个位置。因为所有直线的斜率是相同的，所以考虑将 \(d(u, i)\) 降序排序后从大到小扫，当出现 \(d(v, rk(u, i))>\max_{j=i+1}^n\{d(v, rk(u, j))\}\) 的情况时，更新答案： \[ ans\leftarrow\min(ans, d(u, rk(u, i))+\max_{j=i+1}^nd(v, rk(u, j))+w(u,v)) \]

于是我们找到了图的绝对中心，可以完成 CF266D BerDonalds 和 SP735 MDST - Minimum Diameter Spanning Tree。

const int maxn = 1005;
int n, dis[maxn][maxn], rk[maxn][maxn];
vector<int> G[maxn];

void dijkstra(int s, int (&dis)[maxn]) {
    static bool vis[maxn];
    memset(dis, 0x3f, sizeof dis);
    memset(vis, 0, sizeof vis);
    priority_queue<pair<int, int>, vector<pair<int, int> >, greater<pair<int, int> > > q;
    q.push({0, s}), dis[s] = 0;
    while (!q.empty()) {
        int u = q.top().second; q.pop();
        if (vis[u]) continue;
        vis[u] = 1;
        for (auto &v : G[u]) {
            if (dis[u] + 1 < dis[v]) {
                dis[v] = dis[u] + 1;
                q.push({dis[v], v});
            }
        }
    }
    return;
}

int main() {
    int T; read(T);
    while (T--) {
        read(n);
        int ans = 2e9;
        FOR(i, 1, n) {
            vector<int>().swap(G[i]);
            int cnt; read(cnt), read(cnt);
            FOR(j, 1, cnt) {
                int v; read(v);
                G[i].push_back(v);
            }
        }
        FOR(i, 1, n) dijkstra(i, dis[i]);
        FOR(u, 1, n) {
            FOR(j, 1, n) rk[u][j] = j;
            sort(rk[u] + 1, rk[u] + n + 1, [u](const int &a, const int &b) {return dis[u][a] < dis[u][b];});
            chkmin(ans, dis[u][rk[u][n]] << 1);
        }
        FOR(u, 1, n) {
            for (auto v : G[u]) {
                int pos = n;
                DEC(j, n - 1, 1) {
                    if (dis[v][rk[u][j]] > dis[v][rk[u][pos]]) {
                        chkmin(ans, dis[u][rk[u][j]] + dis[v][rk[u][pos]] + 1);
                        pos = j;
                    }
                }
            }
        }
        print(ans);
    }
    return output(), 0;
}

然后我们容易知道，MDST 的直径中点就是图的绝对中心，要构造这样一个生成树，只需要从绝对中心出发然后构造一棵最短路树即可。

支配树

Prufer 序列

前言

将 \(n\) 个节点的无根树与长度为 \(n-2\)，值域 \([1, n]\) 的序列建立双射。

支持 \(O(n)\) 从树构造 Prufer 序列，也支持 \(O(n)\) 从序列还原树。

构造

每次选择一个编号最小的叶节点并删掉他，然后在序列中记录下它连接到的那个点，重复 \(n-2\) 次之后就只剩下一个点。

用堆可以轻松 \(O(n\log n)\)。使用一个指针维护最小值可以做到 \(O(n)\)。具体地，记录一个指向编号最小的叶子节点，每次删掉它之后，如果产生了新的叶子节点并且编号比指针指向的更小，则直接继续删掉，否则自增找到下一个编号最小的叶节点。

然后不难发现一个点的度数就是在 Prufer 序列中的出现次数 \(-1\)，所以可以根据 Prufer 序列求出每个点的度。

还原的时候找到一个编号最小的度数为 \(1\) 的节点，将该节点与当前 Prufer 序列遍历到的点相连，然后同时减掉这两个点的度数，类似地如果产生了新的度数为 \(1\) 的点就接着继续搞。

洛谷模板题代码：

#include <bits/stdc++.h>
#define FOR(i, a, b) for (int i = (a); i <= (b); ++i)

using namespace std;
const int maxn = 5e6 + 5;
int n, m, f[maxn], p[maxn], d[maxn];
long long ans = 0;

void solve1() {
    FOR(i, 1, n - 1) cin >> f[i], ++d[f[i]];
    for (int i = 1, j = 1; i <= n - 2; ++i, ++j) {
        while (d[j]) ++j;
        p[i] = f[j];
        while (i <= n - 2 && !--d[p[i]] && p[i] < j) p[i + 1] = f[p[i]], ++i;
    }
    FOR(i, 1, n - 2) ans ^= 1ll * i * p[i];
    return;
}

void solve2() {
    FOR(i, 1, n - 2) cin >> p[i], ++d[p[i]];
    p[n - 1] = n; // 特判
    for (int i = 1, j = 1; i < n; ++i, ++j) {
        while (d[j]) ++j;
        f[j] = p[i];
        while (i < n && !--d[p[i]] && p[i] < j) f[p[i]] = p[i + 1], ++i;
    }
    FOR(i, 1, n - 1) ans ^= 1ll * i * f[i];
    return;
}

int main() {
    ios::sync_with_stdio(false);
    cin >> n >> m;
    if (m == 1) solve1();
    else solve2();
    cout << ans << endl;
    return 0;
}

性质

  • 根据 Prufer 序列的构造过程，一个点在 Prufer 序列中出现次数为其度数 \(-1\)。

  • 最后剩余的两个点中一定有一个的编号为 \(n\)，因为无论如何树中至少存在两个叶子节点，\(n\) 永远不会被删掉，其编号不可能是最小的。

  • \(n\) 个点组成的无根树的数量是 \(n^{n-2}\)，证明。

  • \(n\) 个点组成的有根树的数量是 \(n^{n-1}\)，考虑在上面的基础上定根。

  • \(n\) 个有度数要求 \(d_i\) 的节点的有标号无根树有 \[ \frac{(n-2)!}{\prod(d_i - 1)!} \] 种，由上面的性质不难推出。

应用

感觉 Prufer 序列本身没有什么特别要注意的，主要都是计数。

例题 \(1\)：CF156D Clues

给定 \(1\le n\le 10^5\) 个点，\(0\le m\le 10^5\) 条边的有标号无向图，其有 \(k\) 个连通块，求添加 \(k - 1\) 条边使得整张图联通的方案数，答案对 \(1\le p\le 10^9\) 取模。

先考虑其有 \(k\) 个点，则我们发现必须枚举 \(d_i\) 序列： \[ \sum_{d_i\ge1,\sum_{i=1}^k d_i = 2k-2}\binom{k-2}{d_1 - 1, d_2 - 1,\cdots, d_n - 1}\prod_{i=1}^ns_i^{d_i} \] 考虑多项式定理 \[ (x_1+x_2+\cdots +x_n)^m = \sum_{k_i\ge 0,\sum_{i=1}^nk_i=m}\binom{m}{k_1,k_2,\cdots, k_n}\prod_{i=1}^nx_i^{k_{i}} \] 令 \(e_i = d_i - 1\)，则原式化简为 \[ (s_1+s_2+\cdots s_k)^{k-2}\prod_{i=1}^ks_k = \sum_{e_i\ge 0,\sum_{i=1}^ke_i = k-2}\binom{k-2}{e_1,\cdots, e_k}\prod_{i=1}^ks_k^{e_i} \] 所以答案为 \(n^{k-2}\displaystyle\prod_{i=1}^ks_k\)。
